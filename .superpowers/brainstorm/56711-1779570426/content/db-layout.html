Database Admin — Page Layout

How should the table list and table data be arranged on screen?

A

Two-panel (split view)

Left panel: table list. Right panel: selected table's rows. Both visible at once — click a table name, rows update inline. Feels like a mini DB explorer.

B

Drill-down (two pages)

/database shows a card grid of tables (name + row count). Click a card → navigate to /database/sources which is a full-page data table with CRUD. Back button returns to the table list.

C

Single page with accordion

One page lists all tables. Each table is an expandable section — click to expand and see its rows inline. All tables visible on one scroll, no navigation needed.
